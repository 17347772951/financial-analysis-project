## 项目背景

### 为什么做这个项目

在财务核算 → 财务分析 / 财务 BP 的转型过程中，大多数候选人面临两个痛点：

1. **简历上只有核算经验**，缺少可展示的分析作品
2. **日常工作中接触不到完整的数据分析链路**（取数 → 清洗 → 建模 → 可视化 → 报告）

这个项目用 12 周时间，从零构建一套企业级财务分析系统，把日常工作中学不到的 **SQL 数据仓库、Power BI 仪表盘、财务分析模型、经营分析报告** 一次性补齐，形成可直接写进简历、面试能讲透的完整作品集。

## 核心产出

### SQL 资产（30+ 脚本）
- [ ] 建表脚本（10 张核心表）
- [ ] 通用财务指标查询（营收/成本/毛利/净利润，按期间 & 公司汇总）
- [ ] 制造模块：单位产品成本计算、成本结构分析、库存周转率
- [ ] 贸易模块：多维度毛利分析、客户盈利排名、费用率分析
- [ ] 跨境模块：汇兑损益计算、出口退税测算、汇率敏感性分析
- [ ] 合并模块：集团合并报表、内部交易抵消、多业态对比

### Power BI 资产（6 页面交互仪表盘）
- [ ] 集团财务概览 —— KPI 卡片 + 趋势图 + 子公司贡献度
- [ ] 制造端成本分析 —— 成本结构瀑布图 + 单位成本对比 + 库存周转
- [ ] 贸易端经营分析 —— 毛利热力图 + 客户排名 + 费用趋势
- [ ] 跨境项目分析 —— 项目全周期跟踪 + 汇兑损益 + 退税进度
- [ ] 风险预警监控 —— 应收账龄 + 异常库存 + 费用超支预警
- [ ] 软件子公司专题 —— 研发费用率 + 税务优惠效果

### 分析资产
- [ ] 成本差异分析模型（实际 vs 标准 → 量差 vs 价差）
- [ ] 外汇风险预警模型（汇率波动对项目利润的敏感性分析）
- [ ] 客户信用评分模型（账龄 + 回款 + 规模三维评分）
- [ ] 季度经营分析报告（20 页 PDF，含执行摘要 + 问题诊断 + 改进建议）

## 快速开始

### 环境要求

| 工具           | 版本      | 用途              |
| -------------- | --------- | ----------------- |
| MySQL          | 8.0+      | 数据仓库          |
| DBeaver        | 最新版    | 数据库管理工具     |
| Power BI Desktop | 最新版  | 数据可视化        |
| Python         | 3.9+      | 数据生成脚本      |
| VS Code        | 最新版    | 代码编辑器        |

### 使用步骤

```bash
# 1. 克隆仓库
git clone https://github.com/[你的用户名]/financial-analysis-project.git

# 2. 创建数据库并导入表结构
mysql -u root -p < sql_scripts/00_create_tables.sql

# 3. 生成模拟数据（或直接使用 data_generator/output/ 下的 CSV）
cd data_generator
pip install pandas numpy faker pymysql
python generate_data.py

# 4. 导入 CSV 数据到 MySQL（使用 DBeaver 导入向导 或 LOAD DATA 命令）

# 5. 打开 Power BI 仪表盘
# 用 Power BI Desktop 打开 power_bi/financial_dashboard.pbix
# 刷新数据源连接（如有需要）
```
